**Roadmap and Risks — Kilinochchi District Scout Platform**

5-Phase Roadmap (dates are illustrative; adjust per governance)
- Q1 — Phase 1: Static site launch with seeded content and stakeholder sign-off.
- Q2 — Phase 2: Public scout profiles and search; privacy compliance review.
- Q3 — Phase 3: Authentication, user accounts, guardian linking.
- Q4 — Phase 4: Request and approval workflows; basic analytics.
- Year 2 Q1 — Phase 5: Full management system, bulk imports, advanced reporting.

Milestones
- MVP static deploy, privacy policy acceptance, guardian consent model, audit-ready request workflow.

Risk Register (top risks and mitigations)
- Data misuse (high): risk that PII of minors is exposed.
  - Mitigations: strict public/private data separation; consent capture; automated scanning for PII in data exports; quarterly audits.
- Low adoption (medium): units may not register or prefer manual processes.
  - Mitigations: onboarding workshops, unit-level champions, simple bulk import tools, low-friction workflows.
- Political/organizational resistance (medium): leadership turnover may deprioritize the platform.
  - Mitigations: align with national standards, produce quarterly impact reports, open governance with unit representatives.
- Technical debt (medium): rushed launches produce fragility.
  - Mitigations: enforce code reviews, automated tests, and limit scope creep.
- Security breach (high): attackers target platform for minors' data.
  - Mitigations: pen tests, least-privilege, encryption, logging, incident plan.

Dependencies
- Hosting provider reliability, email/SMS delivery providers, cloud DB availability.

Governance and Funding
- Propose a governance board with district-level, unit-level, and technical representation.
- Budget: phased operational costs — hosting, backups, security audits, and support.

Success Metrics (repeat for clarity)
- Adoption: % units on platform.
- Engagement: % active scouts.
- Operational improvement: time saved per approval.
